Title: Refresh theme presets and defaults

Summary
Introduce a new preset palette inspired by popular editor themes, update the default Slider look-and-feel to VS Code’s Dark+ palette with lean 1 px outlines, and document the refreshed options so teams can start from balanced, accessible color schemes.

What’s included
- Presets: Replace legacy Light/Dark presets with nine curated themes (Dark+ (Default), Monokai Sunset, Nord Night, Solarized Twilight, High Contrast Slate, Arctic Daylight, VS Minimal Light, Solarized Day, Nimbus Mono) and capture per-preset button fill/outline preferences.
- Defaults: Align global CSS variables, demo deck, and sample content with the Dark+ baseline; outline buttons and slides now use 1 px borders by default.
- UI wiring: Ensure Style modal preset buttons apply `btnFill`, `btnBorderWidth`, and text-color mode without forcing thicker outlines when switching modes.
- Docs & assets: Document the preset lineup, refresh screenshots, examples, and SVG/README/config snippets to match the new colors.
- Tests: Adjust Playwright expectations for new defaults and ensure preset/app background regressions remain covered.

Validation (local)
- Unit: PASS (`npm run test:unit`)
- E2E: PASS (`npm run test:e2e`)

Notes
- Manual smoke in a real browser can confirm subjective theme balance (try outline presets and button hover states).
