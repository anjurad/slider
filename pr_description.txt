Title: Externalize config (URL/File/API) + compact UI + docs

Summary
Adds external configuration support to `slider.html` with a compact inline UI in the Style modal, optional persistence, and developer‑friendly APIs.

What’s included
- Load config via URL parameter `?config=<url>` during initialization.
- Import/Export JSON from the Style modal.
- Programmatic updates via `window.postMessage({ type: 'slider.config', action: 'merge|replace', config })`.
- Persist toggle to control saving to `localStorage` (`slideapp.config`).
- UI layout improvements: shorter URL field, proper Import button, inline row.
- Documentation: README External Config section + `docs/external-config.md` with schema and examples.
- Changelog entry `1.1.0`.

Usage examples
- URL param: `slider.html?config=https://example.com/slideapp-config.json`
- Programmatic:
```
window.postMessage({ type: 'slider.config', action: 'merge', config: { primary: '#01B4E1', accent: '#64FFFC' } }, '*');
```

Notes
- Incoming values are validated and sanitized: hex colors → `#rrggbb`, numeric ranges clamped, enums checked.
- Unknown keys are ignored.
